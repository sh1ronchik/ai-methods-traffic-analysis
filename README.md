## 🏗 Архитектура модели

Модель — **Fully Connected Network (FCN)** с residual-блоками.

```
Input (32) 
    │
    ▼
┌─────────────────────────────┐
│  Stem                        │
│  Linear(32 → 256)            │
│  LayerNorm → GELU            │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐  ×4
│  ResidualBlock               │
│  LayerNorm                   │
│  Linear(256 → 512) → GELU   │
│  Dropout(0.1)                │
│  Linear(512 → 256)           │
│  Dropout(0.1)                │
│  + skip connection           │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│  Head                        │
│  LayerNorm → Linear(256 → 1)│
└─────────────────────────────┘
    │
    ▼
Output (предсказание зарплаты)
```

**Количество параметров:** 1 063 425

### Ключевые архитектурные решения

| Компонент | Выбор | Причина |
|---|---|---|
| Активация | `GELU` | Гладкая, нет мёртвых нейронов как у ReLU |
| Нормализация | `LayerNorm` | Стабильнее BatchNorm на табличных данных |
| Skip-connections | `ResidualBlock` | Устраняет затухание градиентов |
| Loss-функция | `HuberLoss` | Устойчив к выбросам зарплат |
| Трансформация таргета | `log1p(y)` | Зарплаты имеют логнормальное распределение |
| Оптимизатор | `AdamW` | L2-регуляризация через `weight_decay` |
| Планировщик LR | `OneCycleLR` | Быстрый прогрев + косинусный спад |

## 🚀 Быстрый старт

```python
trainer = FCNTrainer(
    x_path="data/X.npy",
    y_path="data/y.npy",
    model_name="lastname_firstname_fcn",   # ← ваше ФИО латиницей
    mlflow_uri="http://kamnsv.com:55000/",
    experiment_name="LIne Regression HH",
    hidden_dim=256,
    n_blocks=4,
    dropout=0.1,
    lr=1e-3,
    weight_decay=1e-4,
    batch_size=512,
    max_epochs=500,
    patience=30,
)

results = trainer.run()
```

Предсказание на полной выборке:

```python
X_full, _ = trainer.load_data()
trainer.predict_and_save(X_full, output_path="result.npy")
```

## 🔬 Технические решения

### Почему log1p(y)?

Зарплаты имеют сильно скошенное распределение — небольшая часть вакансий с очень высокими зарплатами создаёт огромные градиенты при оптимизации MSE в исходном пространстве. Логарифмирование переводит распределение в нормальное, что стабилизирует обучение.

```
До:  y ∈ [1, 194894]  →  правосторонний скос
После: log1p(y) ∈ [0, 12.18]  →  ~нормальное распределение
```

### Почему HuberLoss, а не MSE?

```
MSE:   L = (y - ŷ)²               → огромный штраф за выбросы
Huber: L = 0.5·(y-ŷ)²  если |y-ŷ| ≤ δ
           δ·(|y-ŷ| - 0.5·δ)  иначе
```

Даже в log-пространстве остаются выбросы. HuberLoss(delta=1.0) ведёт себя как MSE вблизи нуля и как MAE на больших ошибках.

### Почему OneCycleLR?

```
Эпохи: [  1..50   ] → LR растёт  0.0001 → 0.001   (прогрев)
        [ 50..500  ] → LR падает  0.001  → 0.0     (косинус)
```

Прогрев позволяет модели найти хорошее направление, спад — сойтись в минимум. Работает значительно быстрее, чем `ReduceLROnPlateau`.

### Early Stopping

```
patience = 30  →  остановка, если val_loss не улучшается 30 эпох подряд
```

Фактическая остановка на эпохе 64 вместо 500 — экономия времени и защита от переобучения.
