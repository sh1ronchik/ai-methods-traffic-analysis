|---|---|

| Вывод | Список зарплат `float`, руб. |
| Веса модели | `resources/ridge_model.joblib` |

---

## 🏗 Архитектура модели

Модель — **Ridge Regression** (линейная регрессия с L2-регуляризацией) в связке с `StandardScaler`.

```
x_data.npy
    │
    ▼
┌──────────────────────────┐
│  StandardScaler           │  ← стандартизация признаков (μ=0, σ=1)
└──────────────────────────┘
    │
    ▼
┌──────────────────────────┐
│  Ridge(alpha=α*)          │  ← L2-регуляризация, α подбирается через GridSearch
└──────────────────────────┘
    │
    ▼
predictions (float, руб.)
```

**Подбор гиперпараметров:** `GridSearchCV` + `KFold(n_splits=5, shuffle=True)`

| Параметр | Диапазон | Шаг |
|---|---|---|
| `alpha` | 0.1 — 15.0 | 0.25 |

**Scoring:** `neg_mean_squared_error`, `neg_mean_absolute_error`, `r2`  
**Refit:** по `neg_mean_squared_error`

## 🚀 Быстрый старт

**Предсказание зарплат:**

```bash
python app.py path/to/x_data.npy
```

Выведет список зарплат в рублях:

```
[45000.0, 78000.0, 120000.0, ...]
```

**Переобучение модели:**

```python
from hw5_hh_regression import RidgeTrainer

trainer = RidgeTrainer(
    x_path="data/X.npy",
    y_path="data/y.npy",
    cv_splits=5,
    random_state=42,
    n_jobs=-1,
    refit_on="neg_mean_squared_error",
)

results = trainer.run()
```

Обученная модель автоматически сохраняется в `resources/ridge_model.joblib`.

## 🔬 Технические решения

### Почему Ridge, а не обычная линейная регрессия?

```
OLS:    минимизирует ‖y - Xβ‖²
Ridge:  минимизирует ‖y - Xβ‖² + α·‖β‖²
```

После `ProfileEncoder` в данных появляется множество one-hot колонок, которые могут быть коллинеарны. Ridge штрафует за большие веса и стабилизирует решение при мультиколлинеарности.

### Почему StandardScaler перед Ridge?

Ridge чувствителен к масштабу признаков — без нормализации признаки с большими значениями (например, возраст в годах vs бинарные флаги) получат непропорционально малые веса. `StandardScaler` приводит все признаки к единому масштабу.

### GridSearchCV + KFold

```
KFold(n_splits=5, shuffle=True, random_state=42)
```

Shuffle перемешивает данные перед разбивкой на фолды — защита от случайной кластеризации объектов с похожими зарплатами в одном фолде.

### Сохранение модели в resources/

```python
joblib.dump(pipeline, "resources/ridge_model.joblib")
```

`joblib` сохраняет весь `Pipeline` (scaler + Ridge) единым объектом. При инференсе достаточно загрузить один файл — трансформация и предсказание выполняются в одном вызове `predict()`.
